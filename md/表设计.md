# devices设备表

| 字段          | 作用                                                       |
| ------------- | ---------------------------------------------------------- |
| id            | 设备唯一编号                                               |
| name          | 设备名称                                                   |
| app_version   | sms gateway app版本                                        |
| token         | 设备token的哈希值                                          |
| login         | 设备注册后生成的登录名                                     |
| password_hash | 登陆密码的哈希值                                           |
| enabled       | 管理员是否允许设备工作                                     |
| status        | 设备状态                                                   |
| last_seen_at  | 服务器最后一次收到该设备心跳或请求的时间，用来判断是否在线 |
| registered    | 设备第一次注册成功的时间                                   |
| created_at    | 数据库记录创建时间，通常与第一次注册时间相近               |
| updated_at    | 设备记录最后更新时间，例如更新名称、状态或版本时修改。     |



# sim卡表

| 名称              | 作用                                                         |
| ----------------- | ------------------------------------------------------------ |
| id                | SIM 记录的唯一编号                                           |
| device_id         | SIM 所属设备的 ID                                            |
| sim_type          | SIM 类型，`PHYSICAL` 表示实体 SIM，`ESIM` 表示 eSIM          |
| slot_index        | Android 系统中的卡槽索引，通常从 `0` 开始。例如第一个卡槽是 `0`，第二个卡槽是 `1`。 |
| sim_number        | 当前项目发送短信时使用的 SIM 编号，通常为 `1` 或 `2`。创建短信任务时通过它指定使用哪张卡。 |
| phone_number      | SIM 对应的电话号码。可能为空或只有脱敏号码，不能保证 Android 一定能读取完整手机号。 |
| carrier_ name     | 运营商名称                                                   |
| iccid_hash        | ICCID 是 SIM 卡的唯一识别号                                  |
| esim_profile_name | ``eSIM 配置文件名称，例如 `Work eSIM`。只对 eSIM 有意义，实体 SIM 可以为空。 |
| esim_group_id     | eSIM 配置或分组标识。                                        |
| enabled           | 服务器是否允许使用这张 SIM 发送短信。                        |
| status            | SIM 当前状态，例如 `active`、`inactive`、`disabled`。        |
| last_used_at      | 这张 SIM 最近一次被用于发送短信的时间，可用于轮询和负载均衡。 |
| created_at        | SIM 记录创建时间。                                           |
| updated_at        | SIM 信息最后更新时间。                                       |
| areas             | 业务地区                                                     |

# 会话表

| 名称                   | 作用                                   |
| ---------------------- | -------------------------------------- |
| id                     | 会话唯一 ID                            |
| external_phone_number  | 对方手机号，一个会话只绑定一个号码     |
| contact_id             | 联系人id                               |
| device_id              | 这个会话使用的手机设备                 |
| sim_card_id            | 这个会话使用的实体 SIM 或 eSIM         |
| sim_number             | 下发给 Android 的 SIM 编号             |
| status                 | 会话状态：打开、关闭或归档             |
| unread_count           | 收到但尚未阅读的消息数量               |
| last_message_preview   | 最后一条消息摘要，用于快速显示会话列表 |
| last_message_direction | 最后一条消息是发送还是接收             |
| last_message_at        | 最后一条消息时间，用于会话排序         |
| created_at             | 会话创建时间                           |
| updated_at             | 会话最后更新时间                       |

# 消息表

| 字段                 | 作用                              |
| -------------------- | --------------------------------- |
| id                   | 消息唯一ID                        |
| conversation_id      | 消息所属对话                      |
| direction            | OUTBOUND表示发出，INBOUND表示收到 |
| message_type         | 消息类型，目前只支持sms           |
| text_content         | 消息正文                          |
| from_phone_number    | 发送方号码                        |
| to_phone_number      | 接收方号码                        |
| state                | 当前消息状态                      |
| device_id            | 实际处理消息的手机                |
| sim_card_id          | 实际使用的 SIM/eSIM               |
| with_delivery_report | 是否请求送达报告                  |
| idempotency_key      | 防止业务系统重复创建同一条消息    |
| valid_until          | 消息过期时间                      |
| pulled_at            | 手机拉取任务的时间                |
| sent_at              | 手机提交短信发送的时间            |
| delivered_at         | 收到送达报告的时间                |
| received_at          | 入站短信的接收时间                |
| error_code           | 发送失败代码                      |
| error_message        | 发送失败原因                      |
| metadata             | 订单 ID、用户 ID 等扩展信息       |
| created_at           | 消息创建时间                      |
| updated_at           | 消息最后更新时间                  |

# 状态历史表

| 字段          | 作用                                                        |
| ------------- | ----------------------------------------------------------- |
| `id`          | 状态记录的自增编号。它不是消息 ID。                         |
| `message_id`  | 对应的消息 ID，例如 `msg_001`。一条消息可以有多条状态记录。 |
| `state`       | 消息变化后的状态。                                          |
| `source`      | 是谁产生了这个状态。                                        |
| `reason`      | 状态变化原因，通常用于记录失败原因或系统处理说明。          |
| `occurred_at` | 状态真正发生的时间。                                        |
| `created_at`  | 服务器将这条记录写入数据库的时间。                          |
